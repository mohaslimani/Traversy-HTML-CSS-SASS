Responsiveness

Responsivness is the act of making a web page apdaptable with different usage (device).

adapting ur page to work with different devices, there are different methodes to do it.

  • adapting the meta tag - set the viewport / scale.
    • this is done default with vs-emmet
    • <meta name="viewport" content="width=device-width, initial-scale=1.0">
  • Use fluid widths/heights instead of fixed - use max-width instead of width, and using relative units(em, %).
    • this is just a strategie w sf, i used it in the first project khanez
    • mainly a container class with {max-width: 990px; margin: auto;}
  • Media queries - CSS option to use different css styling for different sizes.
    • here, as u can see the background and content of the page change based on the width and height of the window.
    • also we used two strategies to change it
    • @media {change properties}
    • targeting different css_files for different window size. using the media attribute in the link stylesheet.
  • Using mainly Rem unit over px.
    • Basics
    • Em unit
    • Rem Unit
  • vh & vw units.
    • vh_vw_home_page example
    • vw and vh : stands for viewport height/width of the actual window/device.
    • represents the height and width of the window in 1000 square, 100/100 height/width.
    • u can say it represent the percentage(%) of the window in it.
    • using this can make the page 100% responsive, but it is a harder method.
    • as u can see here vw and vh is the ultimate form of responsivness.
    • zoom in/out wont make a damn change to our page, but changing the device/heightORwidth of the window will change everything.
    • i dont know why and how but the right scroll_bar in window in chrome is considered a part of the page so making a 3 float-boxs with 33.33vw will make 3 perfect aligned boxes in all devices but chrome(because the scroll takes same of the vw).
